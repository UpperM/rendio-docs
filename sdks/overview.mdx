Rendio ships three official SDKs plus a VSCode extension. All three runtime
SDKs share the same taxonomy, retry behaviour, and OpenTelemetry integration
— you can port code between them with minimal friction.

<CardGroup cols={3}>
  <Card title="Node / TypeScript" icon="js" href="/sdks/rendering">

  <Card title="Python" icon="python" href="/sdks/rendering">

## What every SDK does

| | Node | Python | Go |
|---|---|---|---|
| Reads `RENDIO_API_KEY` from env | ✅ | ✅ | ✅ |
| Typed error hierarchy | ✅ | ✅ | ✅ |
| Automatic retries (2 by default) | ✅ | ✅ | ✅ |
| Exponential backoff with jitter | ✅ | ✅ | ✅ |
| Honors `Retry-After` | ✅ | ✅ | ✅ |
| Auto-generates `Idempotency-Key` on POST retries | ✅ | ✅ | ✅ |
| OpenTelemetry trace propagation | ✅ | ✅ | ✅ |
| `trace_id` / `span_id` on errors | ✅ | ✅ | ✅ |

## Shared defaults

| Option | Default | Env variable |
|---|---|---|
| `baseUrl` | `https://api.rendio.dev` | `RENDIO_BASE_URL` |
| API key | *(required)* | `RENDIO_API_KEY` |
| `timeout` (per attempt) | 30&nbsp;s | — |
| `maxRetries` | 2 | — |
| `telemetry` | on | — |

Override any of these at construction time.

## Hello-world, all three

<Tabs>
  <Tab title="Node">
    ```ts
    import Rendio from "rendio";
    const rendio = new Rendio();
    const pdf = await rendio.pdfs.create({ html: "<h1>Hi</h1>" });
    console.log(pdf.url);
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import rendio
    client = rendio.Rendio()
    pdf = client.pdfs.create(html="<h1>Hi</h1>")
    print(pdf.url)
    ```
  </Tab>
  <Tab title="Go">
    ```go
    client, _ := rendio.NewClient()
    pdf, err := client.Pdfs.Create(ctx, rendio.PdfCreateParams{HTML: "<h1>Hi</h1>"})
    if err != nil { log.Fatal(err) }
    fmt.Println(pdf.URL)
    ```
  </Tab>
</Tabs>
