This page tracks user-visible changes to the Rendio API and the official
SDKs. For incident history, see [status.rendio.dev](https://status.rendio.dev).

## Conventions

- **Breaking changes** ship behind a new version prefix — the current
  `/api/v1/` is the only stable API. We do not break existing paths without
  deprecation.
- **Added** — new parameters, endpoints, or fields. Safe to adopt.
- **Changed** — behaviour tweaks that are backward-compatible but may
  affect edge cases.
- **Fixed** — bug fixes.
- **Deprecated** — supported but no longer recommended; will be removed in
  the next major version.

## Staying informed

- **SDK versions** — bumps follow semver. Watch the `rendio` package on
  npm / PyPI / Go modules.
- **Chromium version bumps** — read at render time from
  `pdf.meta.chromiumVersion` (Node/Python) or `X-Chromium-Version` response
  header. See [Chromium versions](/resources/chromium-versions).
- **Incidents & maintenance** — [status.rendio.dev](https://status.rendio.dev).

## Unreleased — April 21, 2026

### Added

- **Private file storage.** New `storage` parameter on `POST /api/v1/render`
  with `visibility: "private"` and configurable `signedUrlTtl`. Private files
  are served via HMAC-signed URLs that expire independently of file retention.
- **`urlExpiresAt` response field.** Both render and files endpoints return
  `urlExpiresAt` for private files — the ISO 8601 timestamp when the signed
  URL expires.
- **`ttl` query parameter on `GET /api/v1/files/{id}`.** Refresh a signed URL
  with a custom TTL without re-rendering.
- **SDK support (Node, Python, Go).** `StorageOptions` in create params,
  `urlExpiresAt` on result types, and `ttl` on `pdfs.get()`.

## Unreleased — April 19, 2026

### Changed

- **SDK base URL convention (breaking for pre-release users).** All three
  Tier-1 SDKs (Node, Python, Go) now own the `/api/v1/*` path suffix. Users
  pass the **origin only** as `baseUrl` / `base_url` / `WithBaseURL`; the
  SDK appends the correct path internally. Matches Stripe / Anthropic /
  OpenAI conventions.
  - Default `baseUrl` changed from `https://api.rendio.dev` (a subdomain
    that was never deployed) to `https://rendio.dev`.
  - SDK request construction changed from `{baseUrl}/v1/render` to
    `{baseUrl}/api/v1/render`, and from `{baseUrl}/v1/files/{id}` to
    `{baseUrl}/api/v1/files/{id}`.
  - Env var name `RENDIO_BASE_URL` is unchanged — only the expected value
    changes. Callers overriding it (dev, self-signed staging, proxies)
    should set it to the origin only.

### Fixed

- **Standard error envelope on unmatched API routes.** Hitting an unknown
  `/api/*` path now returns the canonical
  `{ "error": { "code": "not_found", "message": "Not Found" } }` envelope
  with a fresh `X-Rendio-Request-Id`, instead of the previous
  non-conforming `{ "error": "Not Found" }` string. Security headers
  (`X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`) are now
  applied to this path.
- **Standard error envelope on 405 Method Not Allowed.** Sending a non-POST
  request to `POST /api/v1/render` now returns the canonical envelope with
  code `validation_error`, an `Allow: POST` header, security headers, and
  an `X-Rendio-Request-Id`. Previously returned a non-conforming string
  error body.
- **SDK error-parser hardening against malformed envelopes.** All three
  Tier-1 SDKs now fall through to status-based typed error dispatch when
  the response body does not match the envelope contract (missing, null,
  string, list, number, or empty-object `error` field). Previous behavior:
  - **Python** raised `AttributeError: 'str' object has no attribute
    'get'` — any malformed body crashed the caller.
  - **Go** misclassified a 404 as `*InternalError`, breaking
    `errors.As(err, &notFoundErr)`.
  - **Node** produced an unhelpful `"HTTP 404"` message with code
    `"unknown_error"`.
  After the fix, a 404 always yields `NotFoundError` (or
  `*NotFoundError`), a 500 always yields `InternalError`, etc., regardless
  of body shape. A new regression guard ensures legitimate
  `{"error":{"code":"unknown_error","message":"…"}}` responses still
  surface the real backend message instead of being replaced by the raw
  body snippet.
- **405 Method Not Allowed → `ValidationError`.** All three Tier-1 SDKs
  now map HTTP 405 to the `ValidationError` class and add 405 to the
  non-retryable status set. Previously fell through to `InternalError` and
  was ambiguously retryable.

## v1.0.0 — April 13, 2026

Initial public release.

### API

- `POST /api/v1/render` — HTML / URL → PDF / PNG / JPEG.
- `GET /api/v1/files/{id}` — metadata for a previously rendered file.
- `GET /api/health` — liveness probe.
- URL mode: upload to CDN, return metadata + URL.
- Buffer mode: raw bytes in the response body.
- Page sizes: `A4`, `A3`, `Letter`, `Legal`, `Tabloid`.
- `preview: true` watermark mode (used by the VSCode extension).
- `Idempotency-Key` header for safe POST retries.
- 17 stable error codes. See [Error codes](/api-reference/errors).
- Per-plan quotas, rate limits, and feature gates. See
  [Plans & quotas](/concepts/plans-and-quotas).

### SDKs

- **Node / TypeScript** — `rendio@0.1.0` on npm. Node 18+, Bun, Deno,
  VSCode extension host. Zero runtime dependencies. Built-in retries
  (default 2), OpenTelemetry tracing, typed error hierarchy.
- **Python** — `rendio==0.1.0` on PyPI. Python 3.9+. Same taxonomy as Node.
- **Go** — `github.com/rendio/rendio-go`. Go 1.22+. Same taxonomy.

### VSCode extension

- `rendio.rendio@0.1.0`. Live PDF preview (500 ms debounce) for `.html`,
  `.hbs`, `.tsx`. Companion `<name>.data.json` mock data. Export command.
  Page-size selector in the status bar.
