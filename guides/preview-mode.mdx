---
title: "Preview Mode"
description: "Render watermarked PDFs without consuming quota. Ideal for template development, CI checks, and design review."
---

Pass `preview: true` to any render call and Rendio returns a watermarked PDF without counting the render against your monthly quota.

<Note>
  Preview mode still requires a valid API key. There is no keyless API access.
</Note>

## Constraints

- **`response` must be `"buffer"`** â€” URL-mode preview renders are rejected with `400 validation_error`. The watermarked output is never stored on the CDN.
- **Daily limits** â€” Free plan: 50 preview renders per day. Starter and Pro: unlimited.
- **Feature gates still apply** â€” `source` (URLâ†’PDF), `javascript`, and `tagged` still check your plan. Preview mode only bypasses quota consumption, not plan-level feature restrictions.

## Code examples

<Tabs>
  <Tab title="Node">
    ```ts
    import Rendio from "rendio";
    import fs from "fs";

    const rendio = new Rendio();

    const pdf = await rendio.pdfs.create({
      html: "<h1>Invoice Template Preview</h1>",
      pageSize: "A4",
      preview: true,
      response: "buffer",
    });

    fs.writeFileSync("preview.pdf", pdf.buffer);
    console.log(`Preview rendered: ${pdf.meta.renderTimeMs} ms`);
    ```

## Use cases

**Template development** â€” iterate on HTML/CSS without burning quota. Open the output PDF alongside your editor to check layout, fonts, and page breaks.

**CI/CD preview checks** â€” render a sample document in your CI pipeline on every pull request to catch layout regressions before merging.

**Design review** â€” share a watermarked PDF with stakeholders for sign-off on layout and content before going to production.

**VSCode extension** â€” the [VSCode extension](/dev-prod-parity/live-preview) uses preview mode under the hood for its live preview panel. Every save triggers a new preview render and highlights word-level changes between versions.

## Handling quota limits on the Free plan

If you exceed 50 preview renders in a day on the Free plan, the API returns `429 rate_limit_exceeded`. Switch to a production render (remove `preview: true`) or wait until the next day.

<Tip>
  For heavy template iteration sessions, consider upgrading to Starter â€” preview
  renders become unlimited and the daily limit disappears.
</Tip>
