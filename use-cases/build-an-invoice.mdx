Build a professional invoice PDF with a Handlebars template, render it
through Rendio, and email the CDN link — all in one end-to-end example.

<Steps>
  <Step title="Install">
    ```bash
    npm install rendio
    ```

    Set your key:

    ```bash
    export RENDIO_API_KEY=rk_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
    ```
  </Step>

  <Step title="Template">

  <Step title="Render">
    ```ts
    import { readFileSync } from "node:fs";
    import Rendio, { RendioError } from "rendio";

    const rendio = new Rendio();
    const template = readFileSync("./templates/invoice.html", "utf-8");

    try {
      const pdf = await rendio.pdfs.create({
        html: template,
        data: {
          company: {
            name: "Acme Studio",
            email: "billing@acmestudio.io",
            address: "123 Design Street, Paris, France",
          },
          client: {
            name: "Globex Corporation",
            email: "accounts@globex.com",
            vatNumber: "FR12345678901",
          },
          invoice: {
            number: "2026-042",
            date: "April 13, 2026",
            dueDate: "April 27, 2026",
            subtotal: "$3,600.00",
            taxRate: 20,
            tax: "$720.00",
            total: "$4,320.00",
            notes: "Payment due within 14 days. Thank you for your business.",
          },
          items: [
            { description: "UI design — 20 hours", qty: 20, unitPrice: "$80.00", total: "$1,600.00" },
            { description: "Frontend development — 25 hours", qty: 25, unitPrice: "$80.00", total: "$2,000.00" },
          ],
        },
        pageSize: "A4",
      });

      console.log(pdf.url);
      console.log(`${pdf.pages} page, ${(pdf.size / 1024).toFixed(1)} KB`);
    } catch (err) {
      if (err instanceof RendioError) {
        console.error(`[${err.code}] ${err.message} (request ${err.requestId})`);
      }
      throw err;
    }
    ```
  </Step>

  <Step title="Email it">
    For a link (PDF hosted on Rendio's CDN for 24h on Free, 30d on paid):

    ```ts
    import { Resend } from "resend";
    const resend = new Resend(process.env.RESEND_API_KEY);

    await resend.emails.send({
      from: "billing@acmestudio.io",
      to: "accounts@globex.com",
      subject: "Invoice #2026-042 — Acme Studio",
      html: `<p>Your invoice is ready.</p><p><a href="${pdf.url}">Download invoice</a></p>`,
    });
    ```

    For a direct attachment (no CDN dependency), re-render with
    `response: "buffer"` and attach the bytes:

    ```ts
    const result = await rendio.pdfs.create({
      html: template,
      data: invoiceData,
      response: "buffer",
    });

    await resend.emails.send({
      from: "billing@acmestudio.io",
      to: "accounts@globex.com",
      subject: "Invoice #2026-042",
      attachments: [{ filename: "invoice.pdf", content: Buffer.from(result.buffer) }],
    });
    ```
  </Step>
</Steps>

## Production checklist

- **Idempotency key** — `idempotencyKey: \`invoice-${invoice.id}-v${invoice.version}\`` so retries don't double-render.
- **Font fallback** — Google Fonts can hiccup. Either pre-fetch or keep `font-family: 'Inter', sans-serif;` so the system `sans-serif` kicks in on miss.
- **Amounts** — Format with `Intl.NumberFormat` in your app, not in Handlebars. Handlebars is dumb about types.
- **Locales** — Pass `invoice.date` and `invoice.dueDate` already-formatted — the template should not do date math.
- **Error branching** — `QuotaExceededError` / `RateLimitError` / `PlanLimitError` each want a different response. See [Error handling](/sdks/error-handling).
